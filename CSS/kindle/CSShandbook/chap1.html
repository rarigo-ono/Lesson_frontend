CSSの基本

CSSの概要

  • ウェブページやアプリケーションスタイル、レイアウト、フォーマットを定義する重要な言語
  • Cascading Style Sheetの略
  • 見た目やデザインを適用するために使用

CSSの役割

  • スタイリング
  • レイアウト
  • レスポンシブデザイン
  • アニメーションとトランジション

CSSの基本構造

  • プロパティと値のペアの構成

    •         セレクタ{
            プロパティ1: 値1;
            プロパティ2: 値2;
        }

  • セレクタ
    CSSルールが適用されるHTML要素を選択
    要素のタグ名、クラス、ID、属性などで指定。
  • プロパティ
    • スタイルを指定する属性（フォント、カラー、マージンなど）を示す
  • 値
    • 適用する具体的な値を指定する(色、サイズ、位置など)

異なるスタイルルールが競合する場合は、特定の要素に適用されるスタイルが決定される。

ボックスモデル

要素をコンテナボックスとして扱い、その中にコンテンツ、余白、ボーダー、パディング、ボックス自体の 幅や高さがなどを定義したモデル

ボックスモデルの要素

  • コンテンツ Content
  • パディング Padding
  • ボーダー Border
  • 余白 Margin

カスケーディング

複数のスタイルルールが同じ要素に適用された場合に、どのスタイルが優先されるか制御する仕組みのこと

原則のルール

  • 重要度(Specificity)
    特定性によって優先順位が決まる。特異性とは、セレクタ内の要素やクラス、IDの数によって計算され、 特定性の高いルールが適用される。
  • 重要なスタイル(!important)
    'important'を追加することで、最優先される。 ただし避けた方が良く、正確なセレクタの仕様と特定性の計算に用いるべき。
  • ソース順序
    HTML文書内で宣言された順序に従い、後に宣言されたルールが前のルールを上書きする。
  • 継承(Inheritance)
    親要素から子要素にスタイルが自動的に伝播するメカニズムのこと。
    一般的に、スタイルは親要素から子要素に伝承される。
    ただし、全てではなく継承可能であるかどうかに依存する。 
        /* 継承の例 */
    .parant {
        font-family: Arial, sans-serif;
        font-size: 16px;
    }

    .child {
        /*font-familyとfont-sizeは親から継承される。
    }

カスケーディングと継承を理解することで、一貫性のあるWEBデザインを実現できる。

ただし、過度に複雑なカスケーディングルールはスタイルの保守性を損なうので、シンプルで効果的なスタイリングが重要。
