STEP1
    デザインカンプを解析して、パーツを抽出する。そしてHTMLでどのようにマークアップするかを検討。
    さらに、レイアウトのポイントや、CSSで定義していく上で注意したい箇所を洗い出す。

    オーソドックスなデザインのコーポレート系サイト
        ページ上部のパーツは、モバイル版とPC版に違いはなく、横幅を伸縮させたデザイン。
        フルードレイアウト(リキッドレイアウト)で作成すればOK。
        フォントサイズと余白の取り方に注意する。
        
        ページ下部のパーツは、モバイル版では縦並びで、PC版では横並びの設定
        フルードレイアウト(リキッドレイアウト)にした上で、並びを変える設計にする。

        ブレークポイントはお任せなので、PC版で横並びにした時に決定する。

    コーディングの基本方針
        モバイルファースト
        パーツ単位で設計（レスポンシブの設定もパーツ単位）

    大きなパーツから抽出
    HTMLのマークアップの検証（できるだけシンプルに、明示的に伝えやすい設計を）
    見出しの階層構造の検証
    クラスによるパーツの区別を設定
    パーツ１    サイト名とナビゲーションメニュー    <header>    class = header
    パーツ２    ヒーローイメージ    <section>   class = hero
                ヒーローのキャッチコピー    <h1>
    パーツ３    メッセージ      <section> class = message
                表題    <h2>
    パーツ４    サービス        <section> class = service
                表題    <h2>
    パーツ５    お問い合わせ    <section> class = contact
                表題    <h2>
    パーツ６    コーポレート情報    <footer> class = footer

    パーツを見せたい流れ
    スクロールに合わせて上から下へ


STEP2
    HTMLとCSSファイルを用意して、基本的な設定を行う。
STEP3
    パーツ単位で作成する。
STEP4
    仕上げと最終チェックを行う。完成ページがレスポンシブになっているかも確認。
    文法チェックや文書構造の確認も忘れずに。
